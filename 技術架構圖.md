# 技術架構圖與選型說明

---

## 整體架構

```
使用者瀏覽器
    │
    ├── 前台（公開）
    │   ├── 首頁、作品、提示詞、技能、工作坊、分享
    │   ├── Google OAuth 登入
    │   └── 會員個人資料頁
    │
    ├── 後台（/admin/*，需登入）
    │   ├── 儀表板、CRUD 管理所有內容
    │   ├── 會員管理
    │   └── 備份與還原
    │
    └── API Routes（伺服器端）
        ├── /api/auth/callback（OAuth 回調）
        ├── /api/backup/cron（定時備份）
        └── /api/backup/manual（手動備份）

         ↓ HTTPS

Vercel（Next.js 託管）
    │
    ├── App Router（Server + Client Components）
    ├── Middleware（路由保護）
    └── Cron Jobs（週期備份觸發）

         ↓

Supabase（後端服務）
    ├── PostgreSQL（12 張資料表）
    ├── Auth（Email/Password + Google OAuth）
    └── Storage（圖片上傳，bucket: assets）

         ↓（備份時）

Gmail SMTP（nodemailer）
    └── 每週自動寄送 JSON 備份到管理員信箱
```

## 資料流

### 前台讀取資料

```
Client Component（'use client'）
    ↓ supabase.from('table').select()
Supabase PostgreSQL（受 RLS 限制）
    ↓ 只返回 status='published' 的資料
前台渲染
```

### 管理員 CRUD

```
管理員登入（Email + Password）
    ↓ JWT 包含 email
後台 CRUD 操作
    ↓ supabase.from('table').insert/update/delete
Supabase 驗證 JWT email = 管理員Email
    ↓ RLS Policy 通過
資料庫執行操作
```

### Google OAuth 登入流程

```
點擊「Google 登入」
    ↓ supabase.auth.signInWithOAuth({ provider: 'google' })
導向 Google 同意頁
    ↓ 使用者授權
Google 導回 Supabase callback URL
    ↓ /api/auth/callback 處理
Supabase 建立 auth.users 記錄
    ↓ 觸發 on_auth_user_created
自動建立 profiles 記錄 + points_log
    ↓
前台顯示登入成功
```

### 備份流程

```
每週日 08:00（台灣）
cron-job.org 發出 GET /api/backup/cron
    ↓ Authorization: Bearer [CRON_SECRET]
Vercel API Route 驗證 token
    ↓ 使用 SUPABASE_SERVICE_ROLE_KEY（bypass RLS）
讀取所有 12 張資料表
    ↓ 整理成 JSON
nodemailer 透過 Gmail SMTP 發送
    ↓ 附件：backup_YYYY-MM-DD.json
管理員信箱收到備份
```

## 資料庫 ER 關係

```
auth.users（Supabase 內建）
    │ 1:1
    └── profiles（使用者個人資料）
            │ 1:N
            └── points_log（積分變動紀錄）

（以下為獨立資料表，無外鍵關係）
projects        → 作品介紹
prompts         → 提示詞
skills          → 技能
workshops       → 工作坊
shared_projects → 社群分享
categories      → 分類（type: skill|prompt|project）
feedback        → 使用者回饋
about_me        → 關於我（單筆）
privacy_policy  → 隱私政策（單筆）
site_settings   → 網站設定（單筆）
```

## vercel.json（Cron Job 設定）

```json
{
  "crons": [
    {
      "path": "/api/backup/cron",
      "schedule": "0 0 * * 0"
    }
  ]
}
```

`0 0 * * 0` = 每週日 UTC 00:00（台灣週日 08:00）

## .env.local 範例（填入你自己的值）

```env
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://你的專案ID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...（你的 anon key）
SUPABASE_SERVICE_ROLE_KEY=sb_secret_...（你的 secret key）

# 備份 Email
GMAIL_USER=你的帳號@gmail.com
GMAIL_PASS=xxxx xxxx xxxx xxxx（Gmail 應用程式密碼，16碼）
BACKUP_EMAIL_TO=你的備份收件信箱@gmail.com

# Cron Job 安全密碼（自行設定一串亂碼）
CRON_SECRET=your_random_secret_key_here
```
