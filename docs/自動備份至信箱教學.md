# 自動每週備份至信箱設定教學（Gmail 版）

本教學使用 **Gmail 應用程式密碼 + nodemailer**，結合 **Vercel Cron Jobs**，實作每週自動備份 Supabase 所有資料表，並把 JSON 備份檔作為附件傳送到您的私人信箱。

---

## 階段一：取得 Gmail 應用程式密碼

> ⚠️ **注意**：這不是你的 Gmail 登入密碼，是 Google 另外產生的 16 碼「應用程式專用密碼」。

1. 登入你的 Google 帳號，前往 **[Google 帳戶安全性設定](https://myaccount.google.com/security)**。
2. 確認已開啟「**兩步驟驗證**」（若未開啟請先開啟，才能繼續下一步）。
3. 在搜尋框輸入「**應用程式密碼**」並點進去。
4. 在「選取應用程式」填入任意名稱，例如：`網站備份服務`，按「**建立**」。
5. 畫面出現一組 **16 碼密碼**（格式如 `xxxx xxxx xxxx xxxx`），複製並保存起來（只顯示一次）。

## 階段二：安裝套件

在專案根目錄的終端機執行：

```bash
npm install nodemailer
npm install --save-dev @types/nodemailer
```

## 階段三：設定環境變數

### 步驟 A：本地開發環境（`.env.local`）

打開專案根目錄的 `.env.local`，加入以下內容：

```env
# Gmail 寄件帳號與應用程式密碼
GMAIL_USER=你的帳號@gmail.com
GMAIL_PASS=xxxxxxxxxxxxxxxx

# 接收備份的目標信箱（可以填同一個 Gmail）
BACKUP_EMAIL_TO=你的接收信箱@gmail.com

# Cron Job 安全密碼（防止被惡意呼叫，隨意填一串亂碼即可）
CRON_SECRET=my_super_secret_backup_key_2026

# Supabase Service Role Key（有權限讀取所有資料表）
# 在 Supabase → Settings → API Keys → Secret keys 找到（新版 Supabase 已更名）
# 舊版顯示為 service_role key（eyJ... 開頭）
# 新版顯示為 Secret key（sb_secret_... 開頭）
# 複製完整 key 貼入即可，兩種格式都有效
SUPABASE_SERVICE_ROLE_KEY=sb_secret_...（你的 Secret key）
```

> 📌 `.env.local` 只用於本地測試，不會被 git 上傳，是安全的。

### 步驟 B：Vercel 線上環境

1. 前往 **Vercel 後台 → 你的專案 → Settings → Environment Variables**。
2. 逐一新增以下 4 個變數（Key 和 Value）：

| Key | Value |
|-----|-------|
| `GMAIL_USER` | 你的帳號@gmail.com |
| `GMAIL_PASS` | 16碼應用程式密碼 |
| `BACKUP_EMAIL_TO` | 收件信箱@gmail.com |
| `CRON_SECRET` | 同 `.env.local` 的亂碼 |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase 的 service_role key |

3. 儲存後，之後每次重新部署這些變數就會自動生效。

## 階段四：備份 API 路由（已建立）

程式碼已建立在 `src/app/api/backup/cron/route.ts`。

**流程說明：**
1. Vercel Cron Job 每週日凌晨 00:00（UTC）自動呼叫此 API。
2. API 先驗證 `Authorization` header 是否符合 `CRON_SECRET`，防止未授權呼叫。
3. 使用 `SUPABASE_SERVICE_ROLE_KEY` 讀取所有資料表內容。
4. 整理成 JSON 檔案作為附件，透過 Gmail（nodemailer）發送到你的信箱。

## 階段五：設定每週自動執行規則

### 方式 A：Vercel 內建 Cron（需 Pro 方案才有完整 UI）

在專案**根目錄**新增（或已存在則編輯）`vercel.json`（已建立）：

```json
{
  "crons": [
    {
      "path": "/api/backup/cron",
      "schedule": "0 0 * * 0"
    }
  ]
}
```

部署後到 Vercel → 專案 → Deployments 確認成功，Cron Jobs 選項若出現代表已啟用。

---

### 方式 B：使用 cron-job.org（免費，推薦）

> 不依賴 Vercel 方案，完全免費，設定簡單。

1. 前往 **[https://cron-job.org](https://cron-job.org)** 註冊帳號。
2. 登入後點 **「Create cronjob」**，在 **COMMON** 頁籤填入：
   - **Title**：網站資料庫備份
   - **URL**：`https://你的網域/api/backup/cron`（例如：`https://ai.huobest.com/api/backup/cron`）
   - **Schedule**：選 **Custom**，設定如下：
     - Days of week：只選 **Sunday**
     - Hours：選 **8**
     - Minutes：選 **0**
     - Days of month、Months：全部保持選取
     - 下方 Crontab expression 應顯示 `0 8 * * 0`，右側 Next executions 確認是週日 08:00
3. 切換到 **ADVANCED** 頁籤：
   - **Request method**：確認是 **GET**
   - **Headers** 區塊，點右側 **「+ ADD」** 按鈕，填入：
     - **Key**：`Authorization`
     - **Value**：`Bearer 你設定的CRON_SECRET`（注意：Value 必須加 `Bearer ` 前綴，例如：`Bearer backup_secret_2026`）
4. 點右下角 **「CREATE」** 儲存。

> ⚠️ **注意**：`.env.local` 和 Vercel 環境變數的 `CRON_SECRET` 只填**純密碼**，不加 `Bearer`。`Bearer ` 只在 cron-job.org 的 Value 欄位加。

設定後，每週日早上 08:00 cron-job.org 會自動呼叫你的 API 觸發備份並發送到信箱。

**Cron 時間說明（方式 A 參考）：**
- `0 0 * * 0` = 每週日凌晨 00:00 UTC，台灣時間週日 08:00
- 若想改成每天備份：`0 0 * * *`
- 若想改成每週一台灣早上 09:00：`0 1 * * 1`

## 階段六：手動測試備份

部署到 Vercel 後，最簡單的測試方式是使用**後台管理頁面**：

1. 前往 `https://你的網域/admin`，以管理員帳號登入。
2. 點選側邊欄「**備份與還原**」。
3. 點擊「**立即備份到信箱**」按鈕。
4. 若出現成功訊息，代表設定正確，信箱應會收到 JSON 附件。

> ⚠️ 若出現 `Missing credentials for "PLAIN"` 錯誤，表示 Vercel 環境變數尚未填入或尚未重新部署。請到 Vercel → Settings → Environment Variables 確認 5 個變數都已填入，再到 Deployments → 最新部署 → 點右側 ⋯ → Redeploy。

7. ✅ 到後台「備份與還原」頁面點「立即備份到信箱」測試
8. ✅ 之後每週日早上 08:00（台灣時間）你的信箱會自動收到備份

> 💡 **SUPABASE_SERVICE_ROLE_KEY 位置**：Supabase → Settings → API Keys → **Secret keys**（新版 Supabase 更名，`sb_secret_...` 開頭即是）
